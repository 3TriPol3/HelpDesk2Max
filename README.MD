### Код в соотвесвии со структурой
#### Технологии
- **Python** - основной язык программирования
- **Peewee** - ORM для подключения к БД
- **MySQL** - ситема управления базами данных
- **PyMySQL** - библиотека для подключения к MySQL
- **bcrypt** - библиотека для хешироваания паролей

#### Модели
### `Models/User.py` — Пользователи системы
Учётные записи сотрудников: авторы заявок, специалисты, администраторы.

| Поле | Тип | Описание |
|------|-----|---------|
| `id` | `PrimaryKeyField()` | Автоинкрементный идентификатор |
| `login` | `CharField(12)` | Уникальный логин (до 12 символов) |
| `password` | `CharField(255)` | Хеш пароля (bcrypt) |
| `role` | `CharField(choices)` | Роль: *Пользователь*, *Специалист*, *Администратор* |
| `is_active` | `BooleanField(default=True)` | Флаг активности (вместо удаления) |
| `fullname` | `CharField(150, null=True)` | Полное имя (необязательно) |

---

### `Models/Category.py` — Категории заявок
Группировка заявок по типу проблемы.

| Поле | Тип | Описание |
|------|-----|---------|
| `id` | `PrimaryKeyField()` | Первичный ключ |
| `name` | `CharField(150, unique=True)` | Название категории (уникальное) |

---

### `Models/Task.py` — Заявки техподдержки
Запросы от пользователей на устранение неполадок.

| Поле | Тип | Описание |
|------|-----|---------|
| `id` | `PrimaryKeyField()` | Идентификатор заявки |
| `topic` | `CharField(100)` | Тема обращения |
| `description` | `TextField()` | Подробное описание проблемы |
| `path` | `CharField(255, null=True)` | Путь к прикреплённому файлу |
| `priority` | `CharField(choices)` | Приоритет: *Низкий*, *Средний*, *Высокий* |
| `status` | `CharField(choices)` | Статус: *Новая*, *В работе*, *Выполнена* и др. |
| `user_id` | `ForeignKeyField(User)` | Автор заявки |
| `specialist_id` | `ForeignKeyField(User, null=True)` | Назначенный специалист |
| `category_id` | `ForeignKeyField(Category)` | Категория проблемы |

---

### `Models/KnowledgeBase.py` — База знаний
Хранение шаблонных вопросов и ответов. Поиск по хешу текста.

| Поле | Тип | Описание |
|------|-----|---------|
| `id` | `PrimaryKeyField()` | Идентификатор записи |
| `query` | `CharField(64, unique=True)` | SHA-256 хеш текста запроса |
| `response` | `TextField()` | Текст ответа |
| `text_query` | `CharField(500)` | Исходный текст вопроса |

#### Контроллеры

### `Controllers/UserController.py`
Управление пользователями: регистрация, обновление, аутентификация.

### `Controllers/CategoryController.py`
Работа с категориями заявок.

### `Controllers/TaskController.py`
Обработка жизненного цикла заявок.

### `Controllers/KnowledgeBaseController.py`
Работа с базой знаний: поиск, добавление, удаление.

## Графический интерфейс

### `Views/AddUserWindow.py`
Модальное окно для регистрации нового пользователя.  
Содержит поля: логин, пароль, подтверждение, роль, ФИО.  
Стиль: тёмная тема (`#2a2d3b`, `#3c415e`), шрифты `Segoe UI`, `Verdana`.

### `Views/EditUserWindow.py`
Редактирование профиля: логин, роль, ФИО.  
Цвета и стиль согласованы с `AddUserWindow`.

### `Views/UserView.py`
Главное окно администрирования:
- Таблица пользователей с прокруткой
- Кнопки: Добавить, Редактировать, Деактивировать, Изменить роль, Обновить
- Статус-бар с количеством загруженных записей

## Точка входа

### `main.py`
Запускает приложение:
1. Проверяет подключение к БД
2. Инициализирует главное окно `UserManagementWindow`
3. Обрабатывает критические ошибки через `messagebox`

---

### Установка
1. Все библиотеки указаны в файле `requirements.txt`
2. Полключение к БД в `Connection/connect.py`
3. Создать таблицы в БД с помощью консоли:
```bash
python Models/create_table.py
```
4. Запуск приложения
